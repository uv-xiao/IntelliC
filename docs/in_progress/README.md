# In-Progress Features

This directory tracks feature branches that are actively being developed.

Rules:
- Each file here represents one feature-sized task, not a small fix.
- Each task must correspond to a GitHub PR based on `htp/feat-*` and targeted at `htp/dev`.
- Creating the task file is the first commit on the branch.
- The task file must stay updated as commits land.
- Before merge, the implementation docs must be moved into `docs/design/`, the TODO entry must be removed or updated, and the task file must be removed from this directory.

## Active tasks

- None currently. Open a new `docs/in_progress/<id>-<topic>.md` file as the first commit of the next feature branch.
