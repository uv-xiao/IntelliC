# Verify Work

Use this before claiming work is complete, fixed, or ready.

## Workflow

1. Identify the strongest available verification command.
2. Run targeted checks for changed contracts.
3. Run broader checks when the repository provides them.
4. Record exact commands and outcomes in the task, PR, or handoff.
5. If a check cannot run, state the concrete blocker and the residual risk.

## Completion Rule

No completion claim is valid without fresh command output.
